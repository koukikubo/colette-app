- 予約管理
- 顧客管理
- 担当者管理
- カレンダーUI

# シリアライザー運用方針 (Serializer Policy)

## 目的

APIのレスポンス形式をシリアライザー（Serializers）を用いて標準化します。

## ゴール

- **オーバーフェッチの防止**: 必要以上のデータ取得を抑制
- **不要なカラムの隠蔽**: セキュリティや設計上の観点から、不要な項目を表示させない
- **フロントエンドとの契約の安定化**: レスポンス形式の一貫性を保ち、予期せぬ変更を防ぐ
- **ドメインロジックと整形処理の分離**: ビジネスロジックとレスポンスの整形処理を切り離す

## ルール

- コントローラーは **ActiveRecordオブジェクトを直接 render してはいけません**。
- JSONレスポンスは必ず **シリアライザーを経由して生成** してください。
- APIのレスポンス形式（Shape）は常に **安定した状態** を維持する必要があります。
